Understanding Phishing and Social Engineering

Phishing is a fraudulent technique where attackers pose as trustworthy sources â€” like a bank or colleague â€” to steal sensitive information. Social engineering manipulates human behavior to bypass technical security controls.

How It Works

  • Fake emails mimicking IT support or legal clients
  • Urgent calls requesting password resets or login confirmation
  • Malicious links disguised as important documents or invoices
Real Case: A lawyer received a fake client email requesting urgent access to documents. The link opened a lookalike login page. Credentials were stolen and used to access confidential case files.

  • Implement anti-phishing tools in email gateways
  • Disable HTML rendering in emails for sensitive accounts

Credential Stuffing and Brute Force (Despite 2FA)

Credential stuffing uses stolen username-password combinations to breach systems. Brute force attacks systematically guess passwords. Even with 2FA, attackers use phishing or SIM-swapping to bypass it.

Attack Vectors

  • Botnets executing large-scale login attempts
  • Phishing pages collecting 2FA tokens directly
  • Malware that intercepts passwords and browser sessions
  • SIM hijacking to access SMS-based 2FA

Prevention Measures

  • Use password managers to generate complex, unique passwords
  • Adopt app-based or biometric 2FA (avoid SMS when possible)
  • Enable geo-location and device-based login alerts
  • Scan the dark web regularly for leaked credentials

Recommended Actions for Next AI Systems

  • Run phishing simulation campaigns every quarter
  • Limit access to legal files using role-based access control (RBAC)
  • Adopt zero-trust network access architecture
  • Log and review admin account activity regularly
  • Deploy adaptive authentication (device history, behavior analytics)
